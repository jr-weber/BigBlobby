Detailed Description

This helper class for BlobMainWindow has functions for controlling the size of windows, widgets, and images.

The public functions of BlobMainWindowView are called on primarily by the View menu at the top of the main window.

void BlobMainWindowView::setFiltersPanelToDefaultFit ( )
slot

Sets the Background, Smoothing, Highpass, and Amplify images to respect a minimum size of MIN_FILTER_IMAGE_WIDTH and MIN_FILTER_IMAGE_HEIGHT and a maximum size of MAX_WIDGET_DIMENSION.

The filters panel is set to respect a minimum size that will accommodate the two images.

The size of the window that holds the filters panel is not affected by this function. If the window happens to be too small to show the minimum sized filters panel, then scroll bars will pop open so that the user can scroll horizontally or vertically to see the images. If the window is larger than the filters panel, then the images will expand to take up as much area as possible while still maintaining a 4:3 aspect ratio. A 4:3 aspect ration was chosen because that is the width to height ratio of a VGA camera image (320 x 240 pixels).

void BlobMainWindowView::zoomFiltersPanelImagesPercent ( int  percent)
slot

Sets the filters panel images (Background, Smoothing, Highpass, and Amplify) to a fixed size that will be the min size (160 x 120 pixels) multiplied by the percent plugged in as an argument.

If the argument is less than 100 percent, then 100 will be used instead. If the argument is greater than (100 * MAX_IMAGE_SCALE_FACTOR), then (100 * MAX_IMAGE_SCALE_FACTOR) will be used.

This function does not affect the size of the window that holds the main panel. If the images are to large too fit in the window, then scroll bars will pop open so that the user can scroll horizontally or vertically as needed to see the images. If the window is larger than needed, the empty area will be filled in with gray.
