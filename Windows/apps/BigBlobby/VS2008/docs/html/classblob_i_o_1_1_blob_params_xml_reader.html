Detailed Description

Knows how to read program settings from an XML file and return a BlobParamsValidator object.

The XML file given to the read() function will be be parsed for the tags seen in the "data/settings/DefaultBlobParams.xml" file. An exception will be thrown if the file cannot be found, if a tag is unrecognized, or if the value for a tag cannot be converted to the correct data type (usually an int or a bool).

Note that calibration settings and camera-specific settings (such as gain and exposure) are stored by CCV in other files that are read elsewhere in the program.

Member Function Documentation

BlobParamsValidator * BlobParamsXmlReader::read ( const std::string &  xmlFile)

Reads and parses an XML file to obtain the program settings that are shown in the GUI.

The Qt QDomDocument class is used for the parsing. See the XML file "data/settings/DefaultBlobParams.xml" for an example of the expected XML tags. If a BlobParamsValidator object is returned successfully, it will hold values that can be used to set the CoreVision object and the GUI controls.

ERROR HANDLING:
This function can throw a FileNotFoundException, a BlobXmlReaderException, an UnknownXmlTagException, or a BlobParamsException. The UnknownXmlTagException is thrown by private helper functions within this class, while the BlobParamsException is thrown by the BlobParamsValidator object when it finds a value that makes no sense (such as a value that cannot be converted into an expected int or bool, or a negative int where only zero or a positive value is acceptable).
