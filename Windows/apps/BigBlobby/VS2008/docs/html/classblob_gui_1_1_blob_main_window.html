Detailed Description

Creates the main window of the Qt GUI placed in front of the CCV 1.3 blob detection code.

The main window is primarily defined in the Qt Designer BlobMainWindow.ui file.

Constructor & Destructor Documentation

BlobMainWindow::BlobMainWindow ( )

The constructor sets up the GUI main window defined in the Qt Designer BlobMainWindow.ui file and then adds to it the GLWidget objects used for rendering the Source image from the video camera, the saved Background image, the filtered images (Smoothing, Highpass, and Amplify) and the final image of the Tracked blobs.

Several helper objects will also be created for specialized tasks like reading/writing settings from XML files, presenting dialog boxes, and adjusting how the main window is viewed. The last object created by the constructor is the BlobSignalsToSlots object, which uses the Qt signals / slots paradigm to make most of the connections from GUI controls to the objects that will call on CCV code to do the actual blob detection.

The constructor also hides a few File menu options that are not quite ready for use ("Load Settings from File..." and "Save Settings to File..."). See the loadSettingsFromOptionalXmlFile() function of the BlobXmlSettings class for an explanation of why "Load Settings from File..." is not ready for use.

void BlobMainWindow::exitWithoutSavingBlobsXml ( )
slot

Calls on the QApplication to terminate the program by closing all windows, but does not save settings to the "data/settings/DefaultBlobParams" XML file.

This slot was added to deal with the special situation where the DefaultBlobParams.xml file is missing, corrupted, or has some kind of invalid contents. In this situation, the user needs to be warned of the problem so that the XML file can be fixed or replaced. However, after warning the user of the problem through a dialog box, if the program exits normally, the DefaultBlobParams.xml file would be overwritten by the normal exit code in the closeEvent() function of this class.

Allowing that overwriting to occur would result in the complete contents of the XML file being overwritten by internal default values from the program (the values set in constructors). Those values might be quite different than the original contents of the DefaultBlobParams.xml that created the error condition, creating a situation where the user might be unclear as to exactly what went wrong and why the error message in the dialog box does not match what is now in the DefaultBlobParams.xml file.

To avoid this odd situation, the dialog box that warns the user of a problem with the XML file should have an Exit button whose signal calls this slot to close the program without touching the DefaultBlobParams.xml file.

void BlobMainWindow::frame ( double  deltaFrameTime)

This is the key function of the class, as it calls on the blob detection and rendering functions.

It starts by calling on the frame() function of the SourceGLWidget to update the CoreVision object and then render the Source image if the camera has a new frame. If a new frame is processed, then this function will also call on the FilterWidgets to render the Background, Smoothing, Highpass, Amplify, and Tracked images.

This function will also check to see if calibration has been requested, in which case it will call on the CalibrationGLWidget to open up. Lastly, this function will check if it is time to update the frames per second and other info shown in the text area of the main window. The time to update is determined in the CoreVision class, and should be about every one second.

void BlobMainWindow::loadDefaultSettingsFromXml ( )

Calls on the BlobXmlSettings object to read general program settings from a default XML file and then use those settings to update both the GUI and the CoreVision object used to access CCV blob detection functionality.

The default XML file is "data/settings/DefaultBlobParams.xml&quot.

ERROR HANDLING:
If an error occurs while attempting to read and parse the default XML file, BlobXmlSettings will warn the user of the error, and then use a set of "factory default" settings that are hard-coded in the BlobXmlValidator class.
