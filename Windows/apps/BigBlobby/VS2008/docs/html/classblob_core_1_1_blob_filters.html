Detailed Description

Provides setter and getter functions for filter parameters along with a draw function for each processed image shown in the GUI (Source, Background, Smoothing, Highpass, Amplify, and Tracked).

The ProcessFilters class of CCV (Community Core Vision) 1.3 provides a single draw function that uses OpenGL to render all images on a single GLUT (OpenGL Utility Toolkit) window. BigBlobby, however, draws each image on a separate QGLWidget object. This subclass of ProcessFilters was written primarily to provide a separate draw function for each image.

BlobFilters::BlobFilters ( )

The constructor initializes the data members of the Filters superclass that are actually used in this subclass.

Other data members are initialized elsewhere in the CCV code. In principle, this initialization should not matter, as the program settings should always be read from an XML file at startup. If the XML file cannot be read, the default constructor of the BlobParamsValidator will provide a set of values.

void BlobFilters::setAmplify ( float  amplify)

Sets the value to be used by the amplify filter.

The value of the pixel will be multiplied by itself and then multiplied by a scaling factor. The scaling factor is the amplify value divided by 128. The max amplify value in the GUI is 300.

Sets the size of the square box used to by the smoothing algorithm.

In the BigBlobby GUI, the scale is from 0 to 15. Behind the scenes, CCV code will double the value and add 1 to make sure that the number of pixels is odd (required by the OpenCV function that does the smoothing).

void BlobFilters::useAutoBackground ( bool  b)

If set to true, the background is dynamically adjusted at a rate set by the setBackgroundLearnRate() function.

This may be useful if their is a problem with environmental lighting changing enough to affect blob detection. When auto background is turned on, each new camera image makes a minor weighted contribution to the saved background image.

void BlobFilters::useDarkBlobs ( bool  b)

If given an arg of true, activates tracking of the dark blobs rather than white blobs.

Front illumination (above the touch surface) would be expected to produce dark blobs, whereas rear illumination (below the touch surface) would be expected to produce white blobs. In the CCV 1.3 GUI, the checkbox labeled INVERSE is used to turn on tracking of dark blobs. In BigBlobby, the "Track Dark Blobs" option in the Blobs menu is used.
