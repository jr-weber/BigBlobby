Detailed Description

Reads/writes XML files for loading/storing the program settings.

In addition to holding a BlobParamsXmlReader and BlobParamsXmlWriter, this class also has a DialogBoxUtils object that it will use to report errors that occur while reading or writing XML.

Constructor & Destructor Documentation

BlobXmlSettings::BlobXmlSettings ( blobGui::BlobMainWindow *  blobMainWindow)

The constructor creates the BlobParamsXmlReader and BlobParamsXmlWriter objects that will be used for reading and writing program settings from/to XML files, as well as a dialog box utility class that will be used for reporting any read/write errors.

The constructor also stores references to key objects that this class will need to use: the GUI main window and the Source GLWidget object that holds onto the CoreVision object used for blob detection.

void BlobXmlSettings::loadDefaultSettings ( )
slot

Loads the default program settings from the file named in DEFAULT_CONFIG_FILE.

This function should always be called when the program first starts up.

ERROR HANDLING:
If the default XML file is missing, corrupted, or has an unrecognized XML tag or an unacceptable value, the user will be informed of the error through a dialog box, and then the "factory default" settings defined in the BlobXmlValidator class will be used instead.

void BlobXmlSettings::loadSettingsFromOptionalXmlFile ( )
slot

WARNING: This function is not ready for use.

Most of the GUI settings can be changed at runtime with no problem, but CCV version 1.3 does not have any provision for changing the camera parameters (width, height, framerate) at runtime. If the XML configuration file read by this function tried to change the camera resolution, the program would not crash, but the camera would remain unchanged. The ability to change camera parameters might be added at a later time, so this function has been left in for possible future use.

Opens a dialog box to ask the user to choose an XML file and then loads program settings from that XML file. Both the GUI and the underlying CoreVision object are updated. This functionality is intended for use by the File menu at the top of the main window.

ERROR HANDLING:
If the requested XML file is missing, corrupted, or has an unrecognized XML tag or an unacceptable value, the user will be informed of the error through a dialog box. The existing program settings will not be changed.

void BlobXmlSettings::saveDefaultSettings ( )
slot

Saves the current program settings to the DEFAULT_CONFIG_FILE.

This function should always be called before the program exits.

NOTE:
The calibration settings and camera-specific settings (gain, exposure, etc.) are not saved here. The calibration.xml file is always saved at the very end of a successful calibration (right after the last point is touched). The cameras.xml file is saved by the destructor of the ofxPS3 class (in the ofxPS3.cpp file).
